# Отчет о доработке системы камеры (WebGL & Standalone)

## Проблематика
1. **WebGL Camera Jumps:** В браузерной версии происходили резкие скачки камеры (телепортация) при вращении мышью, особенно на системах с масштабированием экрана (DPI scaling 125%+).
2. **Jitter & Jagginess:** Движение камеры ощущалось прерывистым ("дерганным") даже при стабильном FPS.
3. **Movement Conflict:** При одновременном движении персонажа и вращении камеры возникала микро-дрожь из-за зависимости расчетов сглаживания от текущего поворота игрока.

## Внесенные изменения

### 1. Фильтрация аномального ввода (Outlier Rejection)
Для WebGL-сборки добавлен фильтр, который игнорирует смещение мыши, если оно превышает 25 единиц за один кадр. Это отсекает технические баги браузеров, передающих некорректные координаты при фиксации курсора.

### 2. Сглаживание вращения (Rotation Smoothing)
Внедрена интерполяция `Mathf.SmoothDampAngle` для углов Yaw и Pitch. 
- Убирает "эффект низкого ФПС" при вращении.
- Параметр сглаживания `CameraSmoothTime` установлен на 0.03с для сохранения высокой отзывчивости.

### 3. Разделение логики по платформам
- **WebGL:** Использует `Input.GetAxis` (Legacy Input) для максимальной стабильности в браузерах.
- **Editor/Standalone:** Использует `_input.look` (New Input System).

### 4. Исправление дрожания при движении
Логика сглаживания полностью переведена на **внутреннее состояние** (`_smoothedYaw`, `_smoothedPitch`). Теперь расчеты не зависят от текущего `transform.rotation` объекта, что устранило конфликт между поворотом персонажа и доводкой камеры.

### 5. Настройка чувствительности
- Введены раздельные множители чувствительности для компенсации сглаживания.
- **Editor/Standalone:** Множитель 2.0.
- **WebGL:** Множитель 1.6 (на 20% ниже для лучшего контроля в браузере).
