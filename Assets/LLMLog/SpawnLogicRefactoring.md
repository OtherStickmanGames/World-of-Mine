# Отчет о рефакторинге логики спавна

## Проблематика
1. **Нарушение принципа единственной ответственности (SRP):** Логика поиска безопасных точек спавна, управления пулом и сетевой синхронизации позиций находилась внутри `NetworkWorldGenerator.cs`.
2. **Засоренность кода:** Генератор мира содержал большой объем кода, не связанного с генерацией чанков, что затрудняло навигацию и поддержку.
3. **"Тихие" ошибки:** Использование защитных проверок на `null` и автоматического поиска зависимостей (`FindObjectOfType`) маскировало ошибки конфигурации в Unity Editor.

## Внесенные изменения

### 1. Выделение NetworkSpawnManager
Создан новый сетевой компонент `NetworkSpawnManager.cs`, полностью отвечающий за жизненный цикл спавна игроков.
- **Pooling:** Реализована фоновая очередь `safeSpawnPool` (5 точек) для мгновенного спавна новых игроков в центре мира.
- **Async Search:** Вынесен алгоритм спиралевидного поиска безопасных точек (`FindSafeSpotNearAsync`) с поддержкой таймаутов и асинхронных пауз (`Task.Yield`) для предотвращения блокировки сервера.

### 2. Оптимизация NetworkWorldGenerator
- Класс полностью очищен от логики управления пулом и RPC-методов спавна.
- Метод `CheckChunkSafetyAsync` сделан публичным, выступая в роли API для проверки "физической" безопасности блоков в чанках.

### 3. Реализация принципа Fail Fast
- Удален весь "защитный" код (null-чеки для инспектора, `FindObjectOfType` как fallback).
- Установлен `[DefaultExecutionOrder(100)]` для гарантированной инициализации синглтона перед обращением к нему.
- Ошибки настройки (неназначенные ссылки) теперь вызывают немедленный `NullReferenceException`, что соответствует стандартам профессиональной разработки.

### 4. Обновление сетевых связей
- `NetworkPlayer.cs` и `PlayerBehaviour.cs` переведены на использование `NetworkSpawnManager.Instance.RequestSafeSpawnServerRpc()`.
- Удалены устаревшие "безопасные" fallback-позиции, маскировавшие отсутствие связи с сервером.

## Файлы

- `Assets/Scripts/Multiplayer/NetworkWorldGenerator.cs` — очистка и API.

- `Assets/Scripts/PlayerBehaviour.cs` — обновление вызовов.
