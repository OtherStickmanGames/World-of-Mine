# Отчет о внедрении серверной валидации голосования в новостях

## Проблематика
1. **Уязвимость к накрутке:** Система голосования в опросах (Release Notes Survey) полагалась только на клиентскую логику. Сервер просто инкрементировал или декрементировал счетчик `votes`, не проверяя, кто именно отправляет запрос. Это позволяло любому пользователю накрутить неограниченное количество голосов через прямые вызовы RPC.
2. **Отсутствие привязки к ID:** В отличие от системы лайков построек, опросы не сохраняли идентификаторы проголосовавших, что делало невозможным контроль уникальности голоса на стороне сервера.

## Внесенные изменения

### 1. Расширение структуры данных (Server-side)
- В структуру `NetworkSurveyData` (файл `CustomNetworkData.cs`) добавлено поле `votedPlayerIds` типа `List<string>`.
- Поле помечено как **NO SENDABLE** (не включено в `NetworkSerialize`), что гарантирует его хранение только в серверных JSON-файлах. Это предотвращает передачу тяжелых списков ID клиентам, сохраняя при этом полную историю голосующих на сервере.

### 2. Строгая серверная валидация
- В `ReleaseNotesHandler.cs` полностью переписана логика `SendSurveySelectServerRpc` и `SendSurveyDeselectServerRpc`.
- **Идентификация:** Для каждого запроса сервер извлекает уникальный `ygPlayerID` (Android Device ID или Yandex ID) через `NetworkUserManager.Instance.playerIds`.
- **Логика Select:** Голос засчитывается только в том случае, если ID игрока отсутствует в списке `votedPlayerIds` выбранного варианта. Счетчик `votes` теперь всегда синхронизируется с фактической длинной списка ID (`votedPlayerIds.Count`).
- **Логика Deselect:** При отмене голоса сервер проверяет наличие ID в списке и удаляет его, корректируя счетчик.
- **Безопасность:** Добавлены проверки на существование новости, наличие массива опроса и корректность индекса варианта, что предотвращает краши при получении некорректных данных.

### 3. Обратная совместимость
- Система автоматически инициализирует пустые списки при загрузке старых JSON-файлов новостей, где поле `votedPlayerIds` отсутствовало.

## Файлы
- `Assets/Scripts/Multiplayer/CustomNetworkData.cs` — обновление структуры данных.
- `Assets/Scripts/Multiplayer/Release Notes/ReleaseNotesHandler.cs` — внедрение логики валидации и синхронизации с JSON.

## Результат
Система голосования стала защищенной от простых методов накрутки. Теперь каждый игрок может отдать только один голос за конкретный вариант ответа, и этот контроль полностью осуществляется на стороне сервера.
