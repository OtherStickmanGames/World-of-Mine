# Отчет об исправлении ввода и чувствительности UI для WebGL

## Проблематика
1. **Нежелательное вращение камеры:** При включении курсора (клавиша F) игрок мог случайно вращать камеру во время взаимодействия с интерфейсом. Это создавало дискомфорт, так как ввод мыши для UI "просачивался" в контроллер игрока, особенно в WebGL-сборках.
2. **Низкая скорость прокрутки в WebGL:** В браузерных сборках стандартная чувствительность `ScrollRect` в админ-панели была крайне низкой из-за специфики обработки `scroll delta` браузерами.

## Внесенные изменения

### 1. Блокировка вращения камеры при активном курсоре
- В `ThirdPersonController.cs` (метод `LateUpdate`) добавлена проверка состояния `_input.cursorInputForLook`.
- Теперь вращение камеры полностью отключается, когда курсор разблокирован для взаимодействия с интерфейсом. Это предотвращает "дрейф" камеры при кликах в меню.

### 2. Корректировка чувствительности ScrollRect
- В `AdminDashboardUI.cs` добавлена логика автоматического повышения чувствительности прокрутки для WebGL-платформы.
- Множитель установлен на **30x** от стандартного значения, что компенсирует малые значения дельты прокрутки в браузерах.
- Изменения изолированы с помощью директив `#if UNITY_WEBGL && !UNITY_EDITOR`, чтобы не нарушать баланс в редакторе Unity.

## Файлы
- `Assets/StarterAssets/ThirdPersonController/Scripts/ThirdPersonController.cs` — логика блокировки вращения.
- `Assets/Scripts/UI/AdminDashboard/AdminDashboardUI.cs` — множитель чувствительности прокрутки.

## Результат
Взаимодействие с админ-панелью и другими элементами интерфейса стало более стабильным и отзывчивым в WebGL-версии игры. Камера больше не реагирует на движения мыши при открытых меню, а списки прокручиваются с ожидаемой скоростью.
