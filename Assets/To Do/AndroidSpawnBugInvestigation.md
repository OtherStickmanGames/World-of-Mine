# Расследование бага спавна на Android (Координата -125, 86, 1)

## Описание проблемы
Игрок на нативной Android-сборке после удаления и повторной установки приложения всегда появляется в одной и той же точке в небе `(-125, 86, 1)`, даже если перед удалением он переместился в другое место. При этом мир вокруг (ямы, постройки) сохраняется корректно.

## Причина (Root Cause)
Проблема вызвана работой системы **Android Auto Backup**:
1. **Механизм:** Android автоматически копирует данные из папки `Application.persistentDataPath` (где лежит `UserData.json`) в облако Google Drive (обычно раз в сутки при зарядке и Wi-Fi).
2. **Цикл бага:** 
   - В облаке сохранился старый бекап с координатой `-125, 86, 1`.
   - Игрок бегает по миру, данные в телефоне обновляются, но в облако не успевают отправиться.
   - При удалении игры локальные (новые) данные стираются.
   - При переустановке Android автоматически восстанавливает "проверенный" файл из облака со старой координатой.
3. **Логика игры:** `PlayerBehaviour.SetLoadedPosition` видит восстановленный файл, считывает из него старую координату (не равную нулю) и пропускает алгоритм безопасного спавна `NetworkSpawnManager`, телепортируя игрока в небо.

## Предложенное решение
Чтобы гарантировать сброс позиции при переустановке, необходимо отключить систему автоматического резервного копирования в манифесте приложения.

### Шаги реализации:
1. Создать (или изменить существующий) файл `Assets/Plugins/Android/AndroidManifest.xml`.
2. В теге `<application>` установить атрибуты:
   ```xml
   android:allowBackup="false"
   android:fullBackupContent="false"
   ```
3. Это заставит Android полностью удалять все данные приложения при деинсталляции без возможности их скрытого восстановления из облака.
